# 📸 Kintone Screenshot Uploader

スクリーンショットをキントーン（職遂/Work アプリ）のサブテーブルに自動登録するChrome拡張機能。

---

## インストール手順

### 1. ファイルを展開
ZIPを解凍して任意のフォルダに配置（例: `C:\Users\tbass\Documents\kintone-uploader\`）

### 2. Chromeに読み込む
1. Chromeのアドレスバーに `chrome://extensions/` を入力してEnter
2. 右上の **「デベロッパーモード」をON**（スライドボタン）
3. **「パッケージ化されていない拡張機能を読み込む」** をクリック
4. 解凍したフォルダを選択して「フォルダーの選択」

### 3. 初回設定
1. Chromeツールバーの 📸 アイコンを右クリック →「オプション」
2. 以下を入力して「設定を保存」：
   - **ログインID**: キントーンのログインID
   - **パスワード**: キントーンのパスワード
   - **従業員番号**: 自分の従業員番号を入力
3. 「接続テスト」で確認

---

## 使い方

1. 記録したいページを開いた状態でツールバーの 📸 アイコンをクリック
2. 現在のタブのスクリーンショットが自動で撮影・表示される
3. 以下を入力：
   - **実施日**: 作業実施日（今日の日付が自動入力）
   - **職遂 / Work**: 作業内容のタイトル
   - **説明 / Explain**: 詳細（任意）
4. 「キントーンに登録」ボタンをクリック
5. 当月の自分のレコードにサブテーブル行が追加されます ✅

---

## フィールドコードの確認・変更方法

キントーンの実際のフィールドコードが異なる場合は `popup.js` の冒頭を修正：

```javascript
const FIELDS = {
  employeeId: "Employee_ID",   // 従業員番号フィールドのコード
  categories: "Categories",    // 月カテゴリのフィールドコード
  subtable: "Table",           // サブテーブルのフィールドコード
  subDate: "Date",             // サブ行の実施日
  subWork: "Work",             // サブ行の職遂
  subExplain: "Explain",       // サブ行の説明
  subEvidence: "Evidence"      // サブ行の証拠（添付）
};
```

フィールドコードの確認方法：キントーンアプリ管理 → フォーム → 各フィールドの歯車アイコン → フィールドコードを確認

## トラブルシューティング

| エラー | 原因 | 対処 |
|--------|------|------|
| 「当月のレコードが見つかりません」 | キントーンに当月レコードがない | キントーンで当月レコードを先に作成 |
| 「APIエラー: 401」 | 認証情報が間違い | 設定画面でID/パスワードを確認 |
| 「APIエラー: 403」 | アクセス権限なし | キントーンの権限設定を確認 |
| スクリーンショットが真っ黒 | 別ウィンドウで操作 | 同じウィンドウで操作してください |
